Complete reference for all Model Context Protocol tools exposed by siza-mcp.

## Generation Tools

### `generate_ui_component`

Generate individual UI components with RAG-based design references.

| Parameter           | Type   | Required | Description                                       |
| ------------------- | ------ | -------- | ------------------------------------------------- |
| `component_type`    | string | yes      | Component type (button, card, form, navbar, etc.) |
| `framework`         | string | yes      | Target framework                                  |
| `description`       | string | no       | Natural language description                      |
| `mood`              | string | no       | Design mood                                       |
| `industry`          | string | no       | Industry context                                  |
| `visual_style`      | string | no       | Visual style                                      |
| `component_library` | string | no       | UI library (shadcn, radix, etc.)                  |

**Example prompt:**

```
Generate a notification bell button with a red badge count,
using React + shadcn/ui, premium mood, SaaS industry
```

### `generate_page_template`

Generate complete page templates with ML composition and quality scoring.

| Parameter      | Type   | Required | Description                                                        |
| -------------- | ------ | -------- | ------------------------------------------------------------------ |
| `template`     | string | yes      | Page type (landing, dashboard, auth, pricing, settings, ecommerce) |
| `framework`    | string | yes      | Target framework                                                   |
| `mood`         | string | no       | Design mood                                                        |
| `industry`     | string | no       | Industry context                                                   |
| `visual_style` | string | no       | Visual style                                                       |

### `scaffold_full_application`

Generate full project boilerplate with ML-composed pages.

| Parameter   | Type     | Required | Description      |
| ----------- | -------- | -------- | ---------------- |
| `app_type`  | string   | yes      | Application type |
| `framework` | string   | yes      | Target framework |
| `pages`     | string[] | no       | Pages to include |
| `mood`      | string   | no       | Design mood      |
| `industry`  | string   | no       | Industry context |

### `generate_from_template_pack`

Generate a multi-page application from a template pack.

| Parameter   | Type   | Required | Description                                                  |
| ----------- | ------ | -------- | ------------------------------------------------------------ |
| `pack`      | string | yes      | Pack name (`saas-dashboard`, `startup-landing`, `ecommerce`) |
| `framework` | string | yes      | Target framework                                             |

### `generate_prototype`

Generate interactive prototypes with component library integration.

| Parameter      | Type     | Required | Description                                   |
| -------------- | -------- | -------- | --------------------------------------------- |
| `description`  | string   | yes      | Prototype description                         |
| `framework`    | string   | yes      | Target framework                              |
| `pages`        | string[] | no       | Pages to include in the prototype             |
| `interactions` | string   | no       | Interaction descriptions (hover, click, etc.) |

### `refine_component`

Iteratively improve components based on natural language feedback.

| Parameter      | Type   | Required | Description             |
| -------------- | ------ | -------- | ----------------------- |
| `code`         | string | yes      | Existing component code |
| `instructions` | string | yes      | Refinement instructions |
| `framework`    | string | no       | Target framework        |

## Design Tools

### `generate_design_image`

Generate design mockups and visual references using AI image generation.

| Parameter     | Type   | Required | Description                         |
| ------------- | ------ | -------- | ----------------------------------- |
| `description` | string | yes      | Design description                  |
| `style`       | string | no       | Image style (mockup, wireframe, ui) |
| `dimensions`  | string | no       | Image dimensions                    |

### `fetch_design_inspiration`

Fetch design inspiration from curated sources.

| Parameter  | Type   | Required | Description                              |
| ---------- | ------ | -------- | ---------------------------------------- |
| `query`    | string | yes      | Search query                             |
| `category` | string | no       | Category filter (web, mobile, dashboard) |
| `limit`    | number | no       | Number of results (default: 10)          |

### `analyze_design_references`

Analyze design reference images and extract patterns, colors, and layout information.

| Parameter   | Type     | Required | Description                                              |
| ----------- | -------- | -------- | -------------------------------------------------------- |
| `image_url` | string   | yes      | URL or base64 of the image                               |
| `extract`   | string[] | no       | What to extract (colors, layout, typography, components) |

### `image_to_component`

Convert screenshots, mockups, or wireframes into framework-specific component code.

| Parameter   | Type   | Required | Description                       |
| ----------- | ------ | -------- | --------------------------------- |
| `image`     | string | yes      | URL or base64 of the design image |
| `framework` | string | yes      | Target framework                  |
| `fidelity`  | string | no       | `pixel-perfect` or `structural`   |

## Figma Integration

### `figma_context_parser`

Parse Figma design files and extract component metadata.

| Parameter      | Type     | Required | Description                 |
| -------------- | -------- | -------- | --------------------------- |
| `file_key`     | string   | yes      | Figma file key from URL     |
| `node_ids`     | string[] | no       | Specific node IDs to parse  |
| `access_token` | string   | yes      | Figma personal access token |

### `figma_push_variables`

Push design tokens and variables to Figma.

| Parameter      | Type   | Required | Description                    |
| -------------- | ------ | -------- | ------------------------------ |
| `file_key`     | string | yes      | Figma file key                 |
| `variables`    | object | yes      | Design token variables to push |
| `access_token` | string | yes      | Figma personal access token    |

## Quality and Feedback

### `audit_accessibility`

Audit components for WCAG 2.1 accessibility violations.

| Parameter | Type   | Required | Description                            |
| --------- | ------ | -------- | -------------------------------------- |
| `code`    | string | yes      | Component code to audit                |
| `level`   | string | no       | WCAG level: `A`, `AA` (default), `AAA` |

Returns a detailed report with:

- Color contrast violations
- Missing ARIA attributes
- Keyboard navigation issues
- Screen reader compatibility

### `submit_feedback`

Submit feedback on generated components for ML self-learning.

| Parameter       | Type    | Required | Description                     |
| --------------- | ------- | -------- | ------------------------------- |
| `generation_id` | string  | yes      | ID of the generation to rate    |
| `rating`        | number  | yes      | Quality rating (1-5)            |
| `feedback`      | string  | no       | Written feedback                |
| `accepted`      | boolean | no       | Whether the output was accepted |

### `analyze_component_library`

Analyze existing component libraries for pattern extraction.

| Parameter    | Type     | Required | Description                    |
| ------------ | -------- | -------- | ------------------------------ |
| `library`    | string   | yes      | Library name or URL            |
| `components` | string[] | no       | Specific components to analyze |

## ML and Training

### `analyze_design_image_for_training`

Analyze design images for ML training data extraction.

| Parameter | Type     | Required | Description          |
| --------- | -------- | -------- | -------------------- |
| `image`   | string   | yes      | Image URL or base64  |
| `labels`  | string[] | no       | Expected labels/tags |

### `manage_training`

Manage ML training pipelines and datasets.

| Parameter | Type   | Required | Description                              |
| --------- | ------ | -------- | ---------------------------------------- |
| `action`  | string | yes      | `start`, `stop`, `status`, `export-data` |
| `dataset` | string | no       | Dataset name                             |

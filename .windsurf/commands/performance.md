# Performance Check

Analyze Siza application performance and Core Web Vitals.

## Performance Metrics:
- LCP (Largest Contentful Paint): Target < 2.5s
- FID (First Input Delay): Target < 100ms
- CLS (Cumulative Layout Shift): Target < 0.1
- FCP (First Contentful Paint): Target < 1.8s
- Bundle Size: Target < 200KB compressed

## Analysis Areas:
1. Bundle size and optimization
2. Image optimization and loading
3. Code splitting and lazy loading
4. Database query performance
5. API response times
6. Memory usage patterns
7. Network request optimization

## Tools Used:
- Lighthouse for Core Web Vitals
- Webpack Bundle Analyzer
- Chrome DevTools Performance
- Database query analysis
- API performance monitoring

## Optimization Strategies:
- Dynamic imports for code splitting
- Image optimization with WebP
- Service worker caching
- Database query optimization
- API response caching
- Memory leak detection

## Usage:
/performance check - Full performance audit
/performance bundle - Bundle size analysis
/performance vitals - Core Web Vitals only
/performance database - Database performance check

## Reporting:
- Performance score (0-100)
- Core Web Vitals metrics
- Bundle size breakdown
- Optimization recommendations
- Before/after comparisons

## Alerts:
- Performance regression detection
- Bundle size increase warnings
- Core Web Vitals degradation
- Database slow query alerts
